--------------------------------------------------------------------------------
A. Database Creation:

2. Right-click on Databases -> Create -> Database...
3. Name the database: ecommerce_db and click Save.

B. Raw Data Staging Upload:
To load raw sales data, we resolve permission issues using standard local paths:
1. Create a folder in your C drive: C:/PostgresData/
2. Save your cleaned data CSV file as: C:/PostgresData/sales_data.csv
3. In pgAdmin, right-click on your ecommerce_db -> select Query Tool.
4. Paste and execute the following DDL block to build the staging table and COPY the CSV:

--- COPY CODE START ---
DROP TABLE IF EXISTS staging_sales CASCADE;
CREATE TABLE staging_sales (
    row_id INT,
    order_id VARCHAR(50),
    year INT,
    order_date DATE,
    ship_date DATE,
    ship_mode VARCHAR(50),
    customer_id VARCHAR(50),
    customer_name VARCHAR(150),
    segment VARCHAR(50),
    country VARCHAR(100),
    city VARCHAR(100),
    state VARCHAR(100),
    postal_code INT,
    region VARCHAR(50),
    product_id VARCHAR(50),
    category VARCHAR(100),
    sub_category VARCHAR(100),
    product_name VARCHAR(255),
    sales NUMERIC(12, 4),
    quantity INT,
    discount NUMERIC(5, 2),
    profit NUMERIC(12, 4)
);

SET datestyle = 'ISO, MDY';
TRUNCATE TABLE staging_sales;
COPY staging_sales(
    row_id, order_id, year, order_date, ship_date, ship_mode,
    customer_id, customer_name, segment, country, city, state,
    postal_code, region, product_id, category, sub_category,
    product_name, sales, quantity, discount, profit
)
FROM 'C:/PostgresData/sales_data.csv'
WITH (
    FORMAT csv,
    HEADER true,
    DELIMITER ','
);
--- COPY CODE END ---

7. STEP 5: POWER BI LIVE DASHBOARD INTEGRATION
--------------------------------------------------------------------------------
A. Connection Mode: DirectQuery
Allows real-time aggregations. Power BI queries PostgreSQL live.

B. Relational Schema Setup (Model View):
The model enforces a standard Star Schema layout:
* dim_customers (customer_id) --- 1:N ---> fact_sales (customer_id)
* dim_locations (postal_code) --- 1:N ---> fact_sales (postal_code)
* dim_products (product_id) --- 1:N ---> fact_sales (product_id)

C. DAX Measures Created:
1. Total Sales = SUM(fact_sales[sales])
2. Total Profit = SUM(fact_sales[profit])
3. Total Orders = DISTINCTCOUNT(fact_sales[order_id])
4. Profit Margin % = DIVIDE([Total Profit], [Total Sales], 0)
5. Average Order Value = DIVIDE([Total Sales], [Total Orders], 0)
6. Sales YTD = TOTALYTD([Total Sales], 'fact_sales'[order_date])
7. Sales Growth MoM = DIVIDE([Total Sales] - CALCULATE([Total Sales], DATEADD('fact_sales'[order_date], -1, MONTH)), CALCULATE([Total Sales], DATEADD('fact_sales'[order_date], -1, MONTH)), 0)

8. EXECUTIVE BUSINESS INSIGHTS SUMMARY (TOP 10 INSIGHTS)
--------------------------------------------------------------------------------
1. The 20% Discount Trap: Discount rate has a strong negative correlation (-0.22) with profit. Discounts above 20% destroy profit margins completely.
2. Furniture Margin Crisis: Furniture generates $742k+ in sales but only $18.4k in profit (2.49% net margin) due to aggressive pricing discounts.
3. Technology is the Financial Engine: Technology has the highest net margin (17.40%), yielding $145k+ in profits. Expand inventory here.
4. Sub-Category Profit Drain: Tables are a major loss maker (-$17.7k loss) followed by Bookcases (-$3.4k loss). Set standard discount limits on these.
5. Holiday Season Peak: November and December generate over 35% of annual revenue. Scale inventory and warehouse operations accordingly.
6. Geographic Giants: California and New York generate the highest sales and profits.
7. Loss-Making Territories: Texas (-$25.7k loss) and Ohio (-$16.9k loss) are highly unprofitable, completely driven by aggressive promotional discounting.
8. Shipping Mode Lead Times: Standard Class is the most used (60%) averaging 5 days. Same Day class is expensive but doesn't increase order size.
9. Customer Segments: Corporate and Home Office segments have 5% higher Average Order Value (AOV) than the Consumer segment.
10. Quantity Optimization: The average purchase quantity is 3 items. Bundle deals should only start at quantity > 5 to preserve profitability.

9. TROUBLESHOOTING & FAQ
--------------------------------------------------------------------------------
Q1: ModuleNotFoundError: No module named 'sqlalchemy' in Jupyter?
Fix: Create a new cell in your notebook and run: !pip install sqlalchemy psycopg2-binary, or run in CMD: py -m pip install sqlalchemy psycopg2-binary and restart your VS Code Kernel.

Q2: PostgreSQL COPY command fails due to permission?
Fix: Move sales_clean.csv to a global accessible directory like C:/PostgresData/sales_data.csv. This gives PostgreSQL service system permission to read the file.

Q3: How do I change password in the Notebook?
Fix: Go to cell 3, edit DB_PASSWORD = 'password' to your password. If you don't edit it, the notebook automatically falls back to Excel/CSV and runs successfully!
